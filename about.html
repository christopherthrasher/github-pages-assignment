About This Website

This page has information regarding GitHub Pages, content management systems, Jekyll, and the full-stack development process.

New Terms

Term Definition
GitHub Pages GitHub Pages is a service that hosts websites directly from GitHub repositories.
Content Management System (CMS) A content management system is software that helps users create and publish website content without manually coding every page.
Jekyll Jekyll is a static site generator supported by GitHub Pages that can help build websites from content files and templates.
Static Site Generator A static site generator is a tool that creates HTML webpages from templates and content files.

Full-Stack Development Process

Step 1: Planning
The full-stack development process begins with planning the website. Developers figure out the purpose of the project, decide what pages and features are needed, and create a design for how users will use with the website.

Step 2: Front-End Development
After planning, developers build the front end using HTML, CSS, and JavaScript. The front end is everything the user sees and interacts with, including webpages, buttons, images, navigation menus, and any other visual element.

Step 3: Back-End Development and Deployment
Next, developers build the back end using servers, databases, and APIs to handle data and application logic. Once the website is working, it is tested, deployed to a hosting platform like GitHub Pages, and maintained with updates over time.
